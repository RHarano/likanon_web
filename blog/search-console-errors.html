Search Consoleエラー対策完全ガイド｜典型10パターンと修正方法

Search Consoleで突然出る警告メール「インデックスに登録されていません」「クロールエラー」——意味が分からず放置していませんか？典型10パターンと修正方法を解説します。

インデックス系エラー

1. 「クロール済み - インデックス未登録」

Googleはクロールしたが、価値が低いと判断して登録しなかった状態。

対処：コンテンツを充実、E-E-A-T強化、内部リンク追加

2. 「検出 - インデックス未登録」

URLは見つけたがクロールがまだ。

対処：1〜2週間待つ、または「URL検査」→「インデックス登録をリクエスト」

3. 「重複ページ、ユーザーにより正規ページとして選択されていません」

canonicalがGoogleに正しく伝わっていない。

対処：canonicalタグの確認、自己参照canonicalの設置

クロール系エラー

4. 「404 (Not Found)」

削除したページへのリンクが残っている。

対処：301リダイレクトで新URLへ転送、または内部リンクを修正

5. 「サーバーエラー (5xx)」

サーバー側の問題でクロール失敗。

対処：サーバー会社に連絡、サーバープラン見直し

6. 「リダイレクトエラー」

リダイレクトチェーン・ループの可能性。

対処：301リダイレクト設定を見直し、A→B直接にする

モバイル系エラー

7. 「モバイル ユーザビリティの問題」

  • クリック可能な要素が近すぎる
  • コンテンツの幅が画面より広い
  • テキストが小さすぎる

対処：レスポンシブ設定、フォントサイズ16px以上、タップ領域48px以上

8. 「Core Web Vitalsの問題」

LCP・CLS・INPのいずれかが基準値未達。

対処：画像最適化、CSS最適化、不要JS削除

セキュリティ系

9. 「セキュリティの問題」

マルウェア感染・ハッキング検出。

対処：即座にサーバー会社・専門家へ。再度クリーンインストール検討

10. 「手動による対策」

Googleからのペナルティ通知。

対処：警告内容を確認→違反箇所修正→再審査リクエスト

対処の優先順位

  1. セキュリティ警告（即対応）
  2. 手動対策（即対応）
  3. サーバーエラー（即対応）
  4. 404・リダイレクトエラー（週内）
  5. モバイル・Core Web Vitals（月内）
  6. インデックス未登録（数ヶ月かけて改善）

まとめ

  • Search Consoleの警告は月1で確認
  • セキュリティ・手動対策は即対応
  • 404は301リダイレクトで対処
  • インデックス問題はコンテンツ品質改善で
