サイト移転・URL変更時のSEO対応完全ガイド

サイト移転はSEOの最大の鬼門。間違えると数年積み上げた評価が一夜で吹き飛びます。ドメイン変更・URL構造変更・サーバー移転の正しい手順を解説します。

移転前の準備（必須）

  • 全URLのリスト化（Excel・スプレッドシート）
  • 旧URL→新URLの対応表作成
  • 現状の検索順位を記録（GRC等）
  • Search Console・GA4のバックアップ
  • 主要被リンク元のリスト化

移転当日の手順

  1. 新サイトを公開
  2. 301リダイレクトを全URL設定
  3. 新サイトで動作テスト
  4. Search Consoleに「アドレス変更ツール」で通知
  5. 新サイトでsitemap.xml送信
  6. 主要被リンク元に連絡（可能なら）

移転後の確認

  • 旧URLが正しく301転送されるか
  • 新URLがインデックスされているか
  • Search Consoleでクロールエラー監視
  • 順位の変動を1〜2週間観察

順位下落への対処

移転直後は1〜3週間順位が下がるのは正常。完全回復まで1〜3ヶ月かかります。

下落が続く場合

  • 301リダイレクトの正常性確認
  • 新サイトのスキーマ・タイトルタグ確認
  • サイト内リンクが新URLに修正されているか
  • 慌てず3ヶ月は待つ

絶対やってはいけないこと

  • 301なしで新サイト公開
  • 301を302で代用
  • 大量の404を放置
  • 同時にデザインも大幅変更（影響が分離できない）

まとめ

  • 移転前の準備が9割
  • 全URLマッピング＋301設定
  • Search Consoleアドレス変更通知
  • 順位回復は1〜3ヶ月
  • 同時に他の変更を加えない
